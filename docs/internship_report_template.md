# 实习报告模板：Python 语法知识库 Agent

## 一、项目背景
本项目选择 Python 语法作为专业基础课知识点来源，构建一个轻量化知识库问答 Agent。系统通过 RAG 流程让大模型只基于知识库内容回答问题，适合课程复习、知识点查询和入门学习辅助。

## 二、需求分析
系统需要支持知识点文档导入、文本分块、语义检索、模糊查询、受约束问答、多轮对话记忆和简单交互界面。用户提问时，系统必须先检索知识库片段，再基于片段组织答案；如果知识库没有相关内容，系统应明确拒答。

## 三、总体设计
系统分为文档处理、检索、问答、记忆和界面五个部分。文档处理模块负责读取 Markdown 或文本文件并切分知识块；检索模块负责通过 Embedding 和相似度计算召回片段；问答模块负责构造提示词并调用大模型；记忆模块保存最近多轮对话；界面模块通过 Streamlit 提供上传文档和聊天问答功能。

## 四、核心实现思路
1. 文档加载：读取内置 Python 语法知识库或用户上传文档，统一转换为 LangChain `Document` 对象。
2. 文本分块：使用 LangChain 文本分块器；如果环境未安装拆分器，则使用项目内置简单分块逻辑。
3. 语义检索：将知识块转换为向量，计算问题向量与知识块向量的余弦相似度。
4. 模糊查询：当语义相似度不足时，通过字符串相似度和关键词命中进行补充召回。
5. 约束问答：提示词明确要求大模型只依据知识库片段回答，缺少依据时拒答。
6. 多轮记忆：保存最近若干轮问答，作为后续问题的上下文。

## 五、测试说明
测试使用 pytest 完成，覆盖文档加载、文本分块、检索排序、模糊匹配、多轮记忆、问答约束、LLM 配置容错和 Embedding 封装。测试环境使用 Fake LLM 和本地 Hash Embedding，不依赖真实外部 API。

## 七、运行与测试结果
运行方式：执行 `streamlit run app.py` 启动系统界面。

测试方式：执行 `pytest -q` 运行自动化测试。

测试结论：核心模块能够完成知识库构建、问题检索、受约束回答和异常输入处理。

## 八、总结
本项目完成了一个轻量级 RAG 知识库问答 Agent，能够围绕 Python 语法知识点进行检索增强问答。系统结构清晰、模块职责明确，便于后续扩展到操作系统、计算机网络等其他课程。
